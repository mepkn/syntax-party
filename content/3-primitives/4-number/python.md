Python `int` has arbitrary precision (no overflow). `float` is 64-bit IEEE 754.

```python
integer     = 42
big         = 10 ** 100   # arbitrary precision, no overflow
float_val   = 3.14
scientific  = 1.5e10
complex_val = 3 + 4j

10 // 3     # 3   — floor division
10 / 3      # 3.333...
10 % 3      # 1
2 ** 10     # 1024
```
